Next.js 16 has landed ahead of Next.js Conf 2025. This release fundamentally changes how caching works—Cache Components, Turbopack as default, and the new proxy.ts reshape how we build performant React applications.

## Cache Components: Explicit Caching

The `"use cache"` directive replaces implicit caching. All dynamic code now executes at request time by default—you opt in to caching explicitly:

Enable with `cacheComponents: true` in your next.config.ts. This completes the Partial Pre-Rendering story—mix static and dynamic content with precision.

## New Caching APIs

Three distinct APIs for cache invalidation:

updateTag(\`user-\${userId}\`);

## Turbopack Is Default

Turbopack is now the default bundler with **2-5x faster** builds and **up to 10x faster** Fast Refresh. Opt out with `--webpack` if needed.

## proxy.ts Replaces middleware.ts

The new `proxy.ts` clarifies the network boundary and runs on Node.js runtime. Migration: rename the file, rename the function to `proxy`.

## Breaking Changes

- **Node.js 20.9.0+** required
- **Async APIs** — `cookies()`, `headers()`, `params`, `searchParams` require `await`
- **AMP and `next lint` removed**
- **Parallel routes** require explicit `default.js` files

## Key Takeaways

Next.js 16 trades implicit magic for explicit control. Cache Components, clearer caching APIs, and Turbopack performance make this a significant upgrade—and the shift toward predictability is the right direction.
